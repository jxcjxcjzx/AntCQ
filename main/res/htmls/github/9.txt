<h2><a id="user-content-soundmanager-2-javascript-sound-for-the-web" class="anchor" href="#soundmanager-2-javascript-sound-for-the-web" aria-hidden="true"><span class="octicon octicon-link"></span></a>SoundManager 2: JavaScript Sound for the Web</h2>

<p>By wrapping and extending HTML5 and Flash Audio APIs, SoundManager 2 brings
reliable cross-platform audio to JavaScript.</p>

<h2><a id="user-content-html5-audio-support" class="anchor" href="#html5-audio-support" aria-hidden="true"><span class="octicon octicon-link"></span></a>HTML5 Audio() Support</h2>

<pre>* 100% Flash-free MP3 + MP4/AAC where supported
* Compatible with Apple iPad 3.2, iPhone/iOS 4 and newer
* Fallback to Flash for MP3/MP4 support, as needed
* SM2 API is transparent; HTML5/flash switching handled internally
* HTML5 API support approximates Flash 8 API features
* Some other formats (WAV/OGG) supported via HTML5, depending on browser
* See "useHTML5Audio" property for implementation details</pre>

<h2><a id="user-content-basic-api-features-flash-8" class="anchor" href="#basic-api-features-flash-8" aria-hidden="true"><span class="octicon octicon-link"></span></a>Basic API Features (Flash 8)</h2>

<pre>* Load, stop, play, pause, mute, seek, pan and volume control of sounds from Javascript
* Events: onload, whileloading, whileplaying, onfinish and more
* ID3V1 and ID3V2 tag support for MP3s (title, artist, genre etc.)</pre>

* MPEG-4 (AAC, HE-AAC, H.264) audio support
* "MultiShot" play (layered/chorusing effects)
* Waveform/frequency spectrum data
* Peak (L/R channel volume) data
* Audio buffering state/event handling</pre>

<h2><a id="user-content-compiling-js-builds--nodebug--jsmin-and-flash-components-as2as3-to-swf" class="anchor" href="#compiling-js-builds--nodebug--jsmin-and-flash-components-as2as3-to-swf" aria-hidden="true"><span class="octicon octicon-link"></span></a>Compiling JS builds (-nodebug, -jsmin) and Flash components, AS2/AS3 to SWF</h2>

<p>An Ant build file defines the tasks for compiling JS and SWF components,
useful if you make changes to the SM2 source and want to recompile.
Google's Closure Compiler is used for the JS. AS2 compilation is done
by MTASC, and AS3 is handled by Adobe's Open Source Flex SDK (mxmlc)
compiler. Refer to the build.xml file for compiler downloads and path
definitions.</p>

<h2><a id="user-content-versioning--development-notes" class="anchor" href="#versioning--development-notes" aria-hidden="true"><span class="octicon octicon-link"></span></a>Versioning / Development Notes</h2>

<p>Releases are versioned by date, e.g., V2.97a.20110424 and are tagged as
such.* The latest official release is always on trunk/master. Post-release
development builds may be available on the appropriate +DEV branch, eg.,
V2.97a.20110801+DEV</p>

project. If making a pull request, use the project's current +DEV
development branch as the merge target instead of “master”, if possible
(please and thank-you.)</p>
