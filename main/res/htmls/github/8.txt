<h1><a id="user-content-buttons-20-1" class="anchor" href="#buttons-20-1" aria-hidden="true"><span class="octicon octicon-link"></span></a>Buttons 2.0</h1>

<p>Buttons is a highly customizable production ready mobile web and desktop css button library. Buttons is a free  open source project created using Sass.</p>

<p>Authors <a href="https://twitter.com/alexwolfe">Alex Wolfe</a> and <a href="https://twitter.com/roblevintennis">Rob Levin</a>.</p>

<h2><a id="user-content-showcase-demo" class="anchor" href="#showcase-demo" aria-hidden="true"><span class="octicon octicon-link"></span></a>Showcase Demo</h2>

<p>View the  <a href="http://unicorn-ui.com/buttons/">showcase demo</a> to see the buttons in action. The showcase provides a full list of examples along with code snippets to speed up development.</p>

<p><a href="http://unicorn-ui.com/buttons/"><img src="https://camo.githubusercontent.com/48102475a03abd8baab9cd9466af5267e48e8c17/68747470733a2f2f7777772e64726f70626f782e636f6d2f732f793963626d786d6968367577726d6d2f627574746f6e732d73686f77636173652e706e673f646c3d31" alt="Buttons 2.0 Showcase" title="Buttons 2.0 Showcase" data-canonical-src="https://www.dropbox.com/s/y9cbmxmih6uwrmm/buttons-showcase.png?dl=1" style="max-width:100%;"></a></p>

<h2><a id="user-content-bower-installation" class="anchor" href="#bower-installation" aria-hidden="true"><span class="octicon octicon-link"></span></a>Bower Installation</h2>

<ol>
<li>If you're using <a href="http://bower.io/">Bower</a> you can run  <code>bower install Buttons</code></li>
</ol>

<h2><a id="user-content-transitioning-from-buttons-10-to-buttons-20" class="anchor" href="#transitioning-from-buttons-10-to-buttons-20" aria-hidden="true"><span class="octicon octicon-link"></span></a>Transitioning From Buttons 1.0 to Buttons 2.0</h2>

<p>We've made some major improvements to the Buttons library. In order to integrate buttons into your current project you'll need to make the following changes:</p>

<ol>
<li>Compass has been replaced with <a href="https://github.com/postcss/autoprefixer">autoprefixer</a>. Compass is not recommended but it is still supported.</li>
<li>Button colors are now completely independent (ex. button-primary). We no longer have classes like <code>button-flat-primary</code>, so to achieve this you now simply add <code>button-flat button-primary</code></li>
<li>Buttons styles are now independent (ex. button-flat, button-3d, etc.). You can apply these styles and they will automatically pick up the color attached to the button (ex. button-primary button-3d).</li>
</ol>

<h2><a id="user-content-customize-buttons-recommended-uses-sass--autoprefixer" class="anchor" href="#customize-buttons-recommended-uses-sass--autoprefixer" aria-hidden="true"><span class="octicon octicon-link"></span></a>Customize Buttons (Recommended uses Sass &amp; Autoprefixer)</h2>

<ol>
<li>Clone the Buttons repository.</li>
<li>Make sure you have <a href="http://nodejs.org/">node.js</a> installed.</li>
<li>From the command line <code>cd</code> into the root for the Buttons directory.</li>
<li>Run <code>npm install</code> or <code>sudo npm install</code> (depending on your system permissions).</li>
<li>On the command line run <code>grunt dev</code>; this will open a browser with Buttons.</li>
<li>Locate <strong>scss</strong> in the root directory.</li>
<li>You can modify the _options.scss file where you can customize colors, typography, etc.</li>
<li>Anytime you save your changes the Buttons showcase page will live reload with your changes!</li>
</ol>

<h2><a id="user-content-customize-buttons-with-only-sass-or-compass" class="anchor" href="#customize-buttons-with-only-sass-or-compass" aria-hidden="true"><span class="octicon octicon-link"></span></a>Customize Buttons with only Sass or Compass</h2>

<ol>
<li>Clone the Buttons repo.</li>
<li>Make sure you have Sass installed.</li>
<li>Run <code>npm install</code> from your terminal.</li>
<li>Edit the <code>_options.scss</code> with your own custom values (see example values below).</li>
<li>Buttons now works with or without Compass, so choose one of the following examples accordingly and run from the command line in Buttons's root directory:<br>
For Sass run: <code>$ sass --watch --scss scss/buttons.scss:css/buttons.css</code><br>
For Compass run: <code>$ compass watch</code></li>
<li>The <code>css/buttons.css</code> file should now be updated.</li>
</ol>

<h2><a id="user-content-button-options" class="anchor" href="#button-options" aria-hidden="true"><span class="octicon octicon-link"></span></a>Button Options</h2>

<p>To edit Buttons simply change values within the <code>_options.scss</code> file. After you make your edits recompile your sass file and your changes will get processed.</p>

<ul>
<li><strong>$ubtn:</strong> This prefix stands for Unicorn Button and prevents namespace collisions that could occur if you import buttons as part of your Sass build process. We kindly ask you not to use the prefix $ubtn in your project in order to avoid possilbe name conflicts. Thanks!</li>
<li><strong>$ubtn-namespace:</strong>  Desired CSS namespace for your buttons (default .button)</li>
<li><strong>$ubtn-glow-namespace:</strong> Desired CSS namespace for your glow effect (default .glow)</li>
<li><strong>$ubtn-colors:</strong> List of colors in format like <code>(name, background, color)</code>.</li>
<li><strong>$ubtn-glow-color:</strong> Default glow color (#2c9adb, light blue)</li>
<li><strong>$ubtn-shapes:</strong> List of shapes in format like <code>(square 0px)</code>. You can use Sass maps if you're using 3.3. See <code>_options.scss</code> for details.</li>
<li><strong>$ubtn-sizes:</strong> List of sizes in format like <code>(jumbo 1.5)</code>. You can use Sass maps if you're using 3.3. See <code>_options.scss</code> for details.</li>
<li><strong>$ubtn-bgcolor:</strong> Default button background color (#EEE, light gray)</li>
<li><strong>$ubtn-height:</strong> Default height, also used to calculate padding on the sides (32px)</li>
<li><strong>$ubtn-font-family:</strong>  Default font family</li>
<li><strong>$ubtn-font-color:</strong> Default font color (#666, gray)</li>
<li><strong>$ubtn-font-weight:</strong> Default font weight</li>
<li><strong>$ubtn-font-size:</strong> Default font size (14px). You can also specify a value of <code>inherit</code> and it will be respected.</li>
</ul>

<h2><a id="user-content-excluding-button-types" class="anchor" href="#excluding-button-types" aria-hidden="true"><span class="octicon octicon-link"></span></a>Excluding Button Types</h2>

<p>By default, Buttons will include all  button types. You can exclude types from your compilation by simply removing the corresponding <em>@import</em> statement in the buttons.scss file.</p>

<div class="highlight highlight-shell"><pre>//Example import statement <span class="pl-k">for</span> <span class="pl-smi">3d</span> button.
@import <span class="pl-s"><span class="pl-pds">'</span>types/3d<span class="pl-pds">'</span></span><span class="pl-k">;</span></pre></div>

<p>Remove this statement then recompile to create a build without 3d buttons.</p>

<h2><a id="user-content-browser-support" class="anchor" href="#browser-support" aria-hidden="true"><span class="octicon octicon-link"></span></a>Browser Support</h2>

<p>Buttons works in all modern browsers (Firefox, Chrome, Safari, IE) and gracefully degrades all to Internet Explorer 8.</p>
