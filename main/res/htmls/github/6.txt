      <span class="octicon octicon-book"></span>

    <article class="markdown-body entry-content" itemprop="mainContentOfPage"><h1><a id="user-content-normalizecss-v3" class="anchor" href="#normalizecss-v3" aria-hidden="true"><span class="octicon octicon-link"></span></a>normalize.css v3</h1>

<p>Normalize.css is a customisable CSS file that makes browsers render all
elements more consistently and in line with modern standards.</p>

<p>The project relies on researching the differences between default browser
styles in order to precisely target only the styles that need or benefit from
normalizing.</p>

<p><a href="http://necolas.github.io/normalize.css/latest/test.html">View the test file</a></p>

<h2><a id="user-content-install" class="anchor" href="#install" aria-hidden="true"><span class="octicon octicon-link"></span></a>Install</h2>

<ul>
<li><a href="http://npmjs.org/">npm</a>: <code>npm install --save normalize.css</code></li>
<li><a href="https://github.com/component/component/">Component(1)</a>: <code>component install necolas/normalize.css</code></li>
<li><a href="http://bower.io/">Bower</a>: <code>bower install --save normalize.css</code></li>
<li><a href="https://cdnjs.com/libraries/normalize">cdnjs</a></li>
<li><a href="http://necolas.github.io/normalize.css/latest/normalize.css">Download</a>.</li>
</ul>

<p>No other styles should come before Normalize.css.</p>

<p>It is recommended that you include the <code>normalize.css</code> file as untouched
library code.</p>

<h2><a id="user-content-what-does-it-do" class="anchor" href="#what-does-it-do" aria-hidden="true"><span class="octicon octicon-link"></span></a>What does it do?</h2>

<ul>
<li>Preserves useful defaults, unlike many CSS resets.</li>
<li>Normalizes styles for a wide range of elements.</li>
<li>Corrects bugs and common browser inconsistencies.</li>
<li>Improves usability with subtle improvements.</li>
<li>Explains what code does using detailed comments.</li>
</ul>

<h2><a id="user-content-browser-support" class="anchor" href="#browser-support" aria-hidden="true"><span class="octicon octicon-link"></span></a>Browser support</h2>

<ul>
<li>Google Chrome (latest)</li>
<li>Mozilla Firefox (latest)</li>
<li>Mozilla Firefox ESR</li>

</ul>

<h2><a id="user-content-extended-details" class="anchor" href="#extended-details" aria-hidden="true"><span class="octicon octicon-link"></span></a>Extended details</h2>

<p>Additional detail and explanation of the esoteric parts of normalize.css.</p>

<h4><a id="user-content-pre-code-kbd-samp" class="anchor" href="#pre-code-kbd-samp" aria-hidden="true"><span class="octicon octicon-link"></span></a><code>pre, code, kbd, samp</code></h4>

<p>The <code>font-family: monospace, monospace</code> hack fixes the inheritance and scaling
of font-size for preformated text. The duplication of <code>monospace</code> is
intentional.  <a href="http://en.wikipedia.org/wiki/User:Davidgothberg/Test59">Source</a>.</p>

<h4><a id="user-content-sub-sup" class="anchor" href="#sub-sup" aria-hidden="true"><span class="octicon octicon-link"></span></a><code>sub, sup</code></h4>

<p>Normally, using <code>sub</code> or <code>sup</code> affects the line-box height of text in all
browsers. <a href="http://gist.github.com/413930">Source</a>.</p>

<h4><a id="user-content-svgnotroot" class="anchor" href="#svgnotroot" aria-hidden="true"><span class="octicon octicon-link"></span></a><code>svg:not(:root)</code></h4>

<p>Adding <code>overflow: hidden</code> fixes IE9's SVG rendering. Earlier versions of IE
don't support SVG, so we can safely use the <code>:not()</code> and <code>:root</code> selectors that
modern browsers use in the default UA stylesheets to apply this style. <a href="http://lists.w3.org/Archives/Public/public-svg-wg/2008JulSep/0339.html">SVG
Mailing List discussion</a></p>

<h4><a id="user-content-inputtypesearch" class="anchor" href="#inputtypesearch" aria-hidden="true"><span class="octicon octicon-link"></span></a><code>input[type="search"]</code></h4>

<p>The search input is not fully stylable by default. In Chrome and Safari on
OSX/iOS you can't control <code>font</code>, <code>padding</code>, <code>border</code>, or <code>background</code>. In
Chrome and Safari on Windows you can't control <code>border</code> properly. It will apply
<code>border-width</code> but will only show a border color (which cannot be controlled)
for the outer 1px of that border. Applying <code>-webkit-appearance: textfield</code>
addresses these issues without removing the benefits of search inputs (e.g.
showing past searches).</p>

<h4><a id="user-content-legend" class="anchor" href="#legend" aria-hidden="true"><span class="octicon octicon-link"></span></a><code>legend</code></h4>

<p>Adding <code>border: 0</code> corrects an IE 8â11 bug where <code>color</code> (yes, <code>color</code>) is not
inherited by <code>legend</code>.</p>
