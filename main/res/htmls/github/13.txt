    <h3>
      <span class="octicon octicon-book"></span>
      README.md
    </h3>

    <article class="markdown-body entry-content" itemprop="mainContentOfPage"><p>This is the source repository for Roboto: Googleâs signature family
of fonts, the default font on Android and ChromeOS, and the
recommended font for Googleâs visual language, Material Design.</p>

<p>It also contains the toolchain used in creating Roboto.</p>

<p>The font family supports all Latin, Cyrillic, and Greek characters in
Unicode 7.0, as well as the currency symbol for the Georgian lari, to
be published in Unicode 8.0.</p>

<p>The fonts are currently available in eighteen different styles.</p>

<p><a href="https://www.google.com/fonts/specimen/Roboto">A subset of an earlier version of Roboto</a> is available from Google Fonts, and can be used as a web font.</p>

<h2><a id="user-content-setup" class="anchor" href="#setup" aria-hidden="true"><span class="octicon octicon-link"></span></a>Setup</h2>

<h4><a id="user-content-create-a-clean-directory-for-roboto" class="anchor" href="#create-a-clean-directory-for-roboto" aria-hidden="true"><span class="octicon octicon-link"></span></a>Create a clean directory for roboto:</h4>

<div class="highlight highlight-bash"><pre>mkdir -p <span class="pl-smi">$HOME</span>/roboto-src
<span class="pl-c1">cd</span> <span class="pl-smi">$HOME</span>/roboto-src</pre></div>

<h4><a id="user-content-download-the-code-and-dependencies" class="anchor" href="#download-the-code-and-dependencies" aria-hidden="true"><span class="octicon octicon-link"></span></a>Download the code and dependencies:</h4>

<div class="highlight highlight-bash"><pre>git clone https://github.com/google/roboto.git
git clone https://github.com/behdad/fonttools.git
git clone https://github.com/robofab-developers/robofab.git
git clone https://github.com/typesupply/feaTools.git
git clone https://github.com/typemytype/booleanOperations.git</pre></div>

<p>download <a href="http://cython.org/#download">Cython</a> and extract it into
the current directory. On Ubuntu, Cython can also be downloaded and installed
via:</p>

<div class="highlight highlight-bash"><pre>sudo apt-get install cython</pre></div>

<h5><a id="user-content-for-otf-generation" class="anchor" href="#for-otf-generation" aria-hidden="true"><span class="octicon octicon-link"></span></a>For OTF generation:</h5>

<div class="highlight highlight-bash"><pre>git clone https://github.com/typesupply/ufo2fdk.git</pre></div>

<p>To build the FDK yourself:</p>

<div class="highlight highlight-bash"><pre>git clone https://github.com/adobe-type-tools/afdko.git</pre></div>

<p>download the latest version of Python 2.7
<a href="https://www.python.org/downloads/">here</a> and extract it into the current
directory.</p>

<p>If you're not building the FDK yourself, download the pre-built version
<a href="http://www.adobe.com/devnet/opentype/afdko.html">here</a> and unzip it into the
current directory.</p>

<h5><a id="user-content-for-ttf-generation-on-ubuntu" class="anchor" href="#for-ttf-generation-on-ubuntu" aria-hidden="true"><span class="octicon octicon-link"></span></a>For TTF generation, on Ubuntu:</h5>

<div class="highlight highlight-bash"><pre>sudo apt-get install fontforge python-fontforge</pre></div>

<h5><a id="user-content-for-post-production" class="anchor" href="#for-post-production" aria-hidden="true"><span class="octicon octicon-link"></span></a>For post-production:</h5>

<div class="highlight highlight-bash"><pre>git clone https://github.com/googlei18n/nototools
git clone https://github.com/rougier/freetype-py.git</pre></div>

<p>download the latest tarball release of HarfBuzz
<a href="http://www.freedesktop.org/wiki/Software/HarfBuzz/">here</a> and extract it into
the <strong>home</strong> directory as <code>$HOME/harfbuzz</code> (alternatively, you can download the
latest source from GitHub via
<code>git clone https://github.com/behdad/harfbuzz.git</code>).</p>

<h4><a id="user-content-install-dependencies" class="anchor" href="#install-dependencies" aria-hidden="true"><span class="octicon octicon-link"></span></a>Install dependencies:</h4>

<p>You can install the necessary modules at the sytem level:</p>

<div class="highlight highlight-bash"><pre><span class="pl-c1">cd</span> fonttools
sudo python setup.py install
<span class="pl-c1">cd</span> ../robofab
sudo python setup.py install
<span class="pl-c1">cd</span> ../feaTools
sudo python setup.py install
<span class="pl-c1">cd</span> ../Cython-0.22
sudo python setup.py install
<span class="pl-c1">cd</span> ../booleanOperations/cppWrapper
sudo python setup.py build_ext --inplace
cp pyClipper.so ../Lib/booleanOperations
<span class="pl-c1">cd</span> ..
sudo python setup.py install
<span class="pl-c1">cd</span> ..</pre></div>

<p>Or set <code>$PYTHONPATH</code> locally before running <code>make</code>:</p>

<div class="highlight highlight-bash"><pre>PYTHONPATH=<span class="pl-s"><span class="pl-pds">"</span><span class="pl-smi">$PYTHONPATH</span>:<span class="pl-smi">$HOME</span>/roboto-src/fonttools/Lib<span class="pl-pds">"</span></span>
PYTHONPATH=<span class="pl-s"><span class="pl-pds">"</span><span class="pl-smi">$PYTHONPATH</span>:<span class="pl-smi">$HOME</span>/roboto-src/robofab/Lib<span class="pl-pds">"</span></span>
PYTHONPATH=<span class="pl-s"><span class="pl-pds">"</span><span class="pl-smi">$PYTHONPATH</span>:<span class="pl-smi">$HOME</span>/roboto-src/feaTools/Lib<span class="pl-pds">"</span></span>
PYTHONPATH=<span class="pl-s"><span class="pl-pds">"</span><span class="pl-smi">$PYTHONPATH</span>:<span class="pl-smi">$HOME</span>/roboto-src/booleanOperations/Lib<span class="pl-pds">"</span></span></pre></div>

<h5><a id="user-content-for-otf-generation-1" class="anchor" href="#for-otf-generation-1" aria-hidden="true"><span class="octicon octicon-link"></span></a>For OTF generation:</h5>

<div class="highlight highlight-bash"><pre><span class="pl-c1">cd</span> ufo2fdk
sudo python setup.py install
<span class="pl-c1">cd</span> ..</pre></div>

<p>Or:</p>

<div class="highlight highlight-bash"><pre>PYTHONPATH=<span class="pl-s"><span class="pl-pds">"</span><span class="pl-smi">$PYTHONPATH</span>:<span class="pl-smi">$HOME</span>/roboto-src/ufo2fdk/Lib<span class="pl-pds">"</span></span></pre></div>

<p>If building the FDK yourself, follow the instructions in <code>afdko/FDK/FDK Build Notes.txt</code>:</p>

<div class="highlight highlight-bash"><pre><span class="pl-c1">cd</span> Python-2.7
./configure --prefix=AFDKOPythonBuild
make install
mv AFDKOPythonBuild ../afdko/FDK/Tools/osx/Python
<span class="pl-c1">cd</span> ../fonttools
sudo ../afdko/FDK/Tools/osx/Python/bin/python setup.py install
<span class="pl-c1">cd</span> ../afdko/FDK/Tools/Programs
./BuildAll.sh
<span class="pl-c1">cd</span> ../..
./FinishInstallOSX
<span class="pl-c1">cd</span> ../..</pre></div>

<p>Otherwise:</p>

<div class="highlight highlight-bash"><pre><span class="pl-c1">cd</span> FDK
./FinishInstallOSX
<span class="pl-c1">cd</span> ..</pre></div>

<p>In either case, use whatever install scripts and directory
(<code>FinishInstall[OSX|Linux|Windows.cmd]</code>,
<code>BuildAll[.sh|Linux.sh|.cmd]</code>,
<code>FDK/Tools/[osx|linux|win]</code>) are appropriate for your platform (more
detailed information can be found in <code>FDK/Read_Me_First.html</code>).</p>

<h5><a id="user-content-for-post-production-1" class="anchor" href="#for-post-production-1" aria-hidden="true"><span class="octicon octicon-link"></span></a>For post-production:</h5>

<div class="highlight highlight-bash"><pre><span class="pl-c1">cd</span> <span class="pl-smi">$HOME</span>/harfbuzz
./configure
make
sudo make install
<span class="pl-c1">cd</span> <span class="pl-smi">$HOME</span>/roboto-src/</pre></div>

<p>Install python modules to system:</p>

<div class="highlight highlight-bash"><pre><span class="pl-c1">cd</span> noto
sudo python setup.py install
<span class="pl-c1">cd</span> ../freetype-py
sudo python setup.py install
<span class="pl-c1">cd</span> ..</pre></div>

<p>Or:</p>

<div class="highlight highlight-bash"><pre>PYTHONPATH=<span class="pl-s"><span class="pl-pds">"</span><span class="pl-smi">$PYTHONPATH</span>:<span class="pl-smi">$HOME</span>/roboto-src/noto<span class="pl-pds">"</span></span>
PYTHONPATH=<span class="pl-s"><span class="pl-pds">"</span><span class="pl-smi">$PYTHONPATH</span>:<span class="pl-smi">$HOME</span>/roboto-src/freetype-py<span class="pl-pds">"</span></span></pre></div>

<p>On Ubuntu (or other distributions of GNU/Linux, using the appropriate package
manager), make sure eog is installed:</p>

<div class="highlight highlight-bash"><pre>sudo apt-get install eog</pre></div>

<h2><a id="user-content-dependencies" class="anchor" href="#dependencies" aria-hidden="true"><span class="octicon octicon-link"></span></a>Dependencies</h2>

<p>The Roboto build toolchain depends on:</p>

<ul>
<li>FontTools (<a href="https://github.com/behdad/fonttools">https://github.com/behdad/fonttools</a>)</li>
<li>RoboFab (<a href="https://github.com/robofab-developers/robofab">https://github.com/robofab-developers/robofab</a>)</li>
<li>NumPy and SciPy (<a href="http://www.numpy.org/">http://www.numpy.org/</a>)</li>
<li>feaTools (<a href="https://github.com/typesupply/feaTools.git">https://github.com/typesupply/feaTools.git</a>), for feature handling.</li>
<li>BooleanOperations (<a href="https://github.com/typemytype/booleanOperations">https://github.com/typemytype/booleanOperations</a>), for
glyph overlap removal.

<ul>
<li>(requires Cython to install: <a href="http://cython.org/">http://cython.org/</a>)</li>
</ul></li>
</ul>

<h2><a id="user-content-otf-generation" class="anchor" href="#otf-generation" aria-hidden="true"><span class="octicon octicon-link"></span></a>OTF Generation</h2>

<p>OTF generation depends on:</p>

<ul>
<li>ufo2fdk (<a href="https://github.com/typesupply/ufo2fdk">https://github.com/typesupply/ufo2fdk</a>)</li>
<li>Open-source portions of the AFDKO
(<a href="https://github.com/adobe-type-tools/afdko/releases">https://github.com/adobe-type-tools/afdko/releases</a>)</li>
</ul>

<p>The AFDKO from GitHub can be time consuming to setup. It is easier to just use
the variety which includes closed-source tools
(<a href="http://www.adobe.com/devnet/opentype/afdko.html">http://www.adobe.com/devnet/opentype/afdko.html</a>), though these closed-source
portions are not used to build Roboto.</p>

<h2><a id="user-content-ttf-generation" class="anchor" href="#ttf-generation" aria-hidden="true"><span class="octicon octicon-link"></span></a>TTF Generation</h2>

<p>TTF generation depends on:</p>

<ul>
<li>FontForge (<a href="https://github.com/fontforge/fontforge">https://github.com/fontforge/fontforge</a>)</li>
</ul>

<p>Whose Python interface should be availabe on Ubuntu by default via <code>apt-get
install fontforge python-fontforge</code>.</p>

<h2><a id="user-content-post-production" class="anchor" href="#post-production" aria-hidden="true"><span class="octicon octicon-link"></span></a>Post-Production</h2>

<p>Post-production scripts (most of the code outside of the <code>fontbuild</code> directory,
e.g. for testing output) depend on:</p>

<ul>
<li>The nototools module, installed as part of Noto
(<a href="https://github.com/googlei18n/nototools">https://github.com/googlei18n/nototools</a>)

<ul>
<li>(Noto subsequently depends on HarfBuzz: <a href="https://github.com/behdad/harfbuzz">https://github.com/behdad/harfbuzz</a>)</li>
</ul></li>
<li>freetype-py (<a href="https://github.com/rougier/freetype-py">https://github.com/rougier/freetype-py</a>)</li>
<li>eog (<a href="https://wiki.gnome.org/Apps/EyeOfGnome">https://wiki.gnome.org/Apps/EyeOfGnome</a>)</li>
</ul>
