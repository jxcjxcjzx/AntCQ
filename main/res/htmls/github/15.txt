      README.rst
    </h3>

    <article class="markdown-body entry-content" itemprop="mainContentOfPage"><h1><a id="user-content-moviepy" class="anchor" href="#moviepy" aria-hidden="true"><span class="octicon octicon-link"></span></a>MoviePy</h1>
<p>MoviePy (full documentation <a href="http://zulko.github.io/moviepy/">here</a>) is a Python module for video editing: cuts, concatenations, title insertions, video compositing (a.k.a. non-linear editing), video processing, and creation of custom effects. See the <a href="http://zulko.github.io/moviepy/gallery.html">gallery</a> for some examples of use.</p>
<p>MoviePy can read and write all the most common audio and video formats, including GIF, and runs on Windows/Mac/Linux, with Python 2.7+ and 3. Here it is in action in an IPython notebook:</p>
<a href="https://raw.githubusercontent.com/Zulko/moviepy/master/docs/demo_preview.jpeg" target="_blank"><img alt="[logo]" src="https://raw.githubusercontent.com/Zulko/moviepy/master/docs/demo_preview.jpeg" style="max-width:100%;"></a>
<a name="user-content-example"></a>
<h2><a id="user-content-example" class="anchor" href="#example" aria-hidden="true"><span class="octicon octicon-link"></span></a>Example</h2>
<p>In this example we open a video file, select the subclip between t=50s and t=60s, add a title at the center of the screen, and write the result to a new file:</p>
<pre>from moviepy.editor import *

video = VideoFileClip("myHolidays.mp4").subclip(50,60)

# Make the text. Many more options are available.
txt_clip = ( TextClip("My Holidays 2013",fontsize=70,color='white')
             .set_position('center')
             .set_duration(10) )

result = CompositeVideoClip([video, txt_clip]) # Overlay text on video
result.write_videofile("myHolidays_edited.webm",fps=25) # Many options...
</pre>
<a name="user-content-contribute"></a>
<h2><a id="user-content-contribute-" class="anchor" href="#contribute-" aria-hidden="true"><span class="octicon octicon-link"></span></a>Contribute !</h2>
<p>MoviePy is an open-source software originally written by <a href="https://github.com/Zulko">Zulko</a> and released under the MIT licence. The project is hosted on <a href="https://github.com/Zulko/moviepy">Github</a> , where everyone is welcome to contribute, ask for help or simply give feedback.</p>
<p>You can also discuss about the project on <a href="http://www.reddit.com/r/moviepy/">Reddit</a> or on the mailing list <a href="mailto:moviepy%40librelist.com">moviepy@librelist.com</a> .</p>
<a name="user-content-installation"></a>
<h2><a id="user-content-installation" class="anchor" href="#installation" aria-hidden="true"><span class="octicon octicon-link"></span></a>Installation</h2>
<p>MoviePy depends on the Python modules <a href="http://www.scipy.org/install.html">Numpy</a>, <a href="http://imageio.github.io/">imageio</a>, <a href="https://pypi.python.org/pypi/decorator">Decorator</a>, and <a href="https://github.com/noamraph/tqdm">tqdm</a>, which will be automatically installed during MoviePy's installation. The software FFMPEG should be automatically downloaded/installed (by imageio) during your first use of MoviePy (it takes a few seconds). If you want to use a specific version of FFMPEG, follow the instructions in file <code>config_defaults.py</code>. In case of trouble, provide feedback.</p>
<p><strong>Installation by hand:</strong> download the sources, either on <a href="https://pypi.python.org/pypi/moviepy">PyPI</a> or (if you want the development version) on <a href="https://github.com/Zulko/moviepy">Github</a>, unzip everything in one folder, open a terminal and type</p>
<pre>(sudo) python setup.py install
</pre>
<p><strong>Installation with pip:</strong> if you have <code>pip</code> installed, just type this in a terminal:</p>
<pre>(sudo) pip install moviepy
</pre>
<p>If you have neither <code>setuptools</code> nor <code>ez_setup</code> installed the command above will fail, is this case type this before installing:</p>
<pre>(sudo) pip install ez_setup
</pre>
<a name="user-content-other-optional-but-useful-dependencies"></a>
<h3><a id="user-content-other-optional-but-useful-dependencies" class="anchor" href="#other-optional-but-useful-dependencies" aria-hidden="true"><span class="octicon octicon-link"></span></a>Other optional but useful dependencies</h3>
<p><a href="http://www.imagemagick.org/script/index.php">ImageMagick</a> is not strictly required, only if you want to write texts. It can also be used as a backend for GIFs but you can do GIFs with MoviePy without ImageMagick.</p>
<p>Once you have installed it, ImageMagick will be automatically detected by MoviePy, <strong>except on Windows !</strong>. Windows user, before installing MoviePy by hand, go into the <code>moviepy/config_defaults.py</code> file and provide the path to the ImageMagick binary called convert. It should look like this</p>
<pre>IMAGEMAGICK_BINARY = "C:\\Program Files\\ImageMagick_VERSION\\convert.exe"
</pre>
<p><a href="http://www.pygame.org/download.shtml">PyGame</a> is needed for video and sound previews (useless if you intend to work with MoviePy on a server but really essential for advanced video editing <em>by hand</em>).</p>
<p>For advanced image processing you will need one or several of these packages. For instance using the method <code>clip.resize</code> requires that at least one of Scipy, PIL, Pillow or OpenCV are installed.</p>
<ul>
<li>The Python Imaging Library (PIL) or, better, its branch <a href="http://pillow.readthedocs.org/en/latest/">Pillow</a> .</li>
<li><a href="http://www.scipy.org/">Scipy</a> (for tracking, segmenting, etc.), and can be used for resizing video clips if PIL and OpenCV aren't installed on your computer.</li>
<li><a href="http://scikit-image.org/download.html">Scikit Image</a> may be needed for some advanced image manipulation.</li>
<li><a href="http://sourceforge.net/projects/opencvlibrary/files/">OpenCV 2.4.6</a> or more recent (which provides the package <code>cv2</code>) may be needed for some advanced image manipulation.</li>
</ul>
